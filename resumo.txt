# RESUMO DO DESENVOLVIMENTO - VELMORA

## Data: 05 de Marco de 2026

O projeto evoluiu de um RPG classico para uma base Souls-like com foco em build, equipamentos por slot, requisitos e escalonamento de arma.

### Principais Implementacoes

#### 1. Sistema Souls-like de Atributos e Equipamentos
- Criado `src/lib/soulslike.ts` para centralizar:
  - `equip load` (carga de equipamento)
  - tiers de mobilidade: `light`, `medium`, `heavy`, `overloaded`
  - bonus/penalidade de combate por carga
  - `attack rating` com scaling por atributos
  - penalidade por nao cumprir requisitos de item
- `ArenaTab` passou a usar calculo Souls-like no combate.
- `StatusTab` e `InventoryTab` mostram resumo de build e carga.

#### 2. Itens, Slots e Build Competitiva
- `items.ts` foi expandido com:
  - `weight`
  - `requirements`
  - `scaling`
  - novos tipos de slot
- Slots ativos no personagem:
  - Arma, Escudo, Capacete, Peitoral, Luvas, Calcas, Botas
- Itens novos adicionados para:
  - `helmet` (5)
  - `gloves` (5)
  - `legs` (5)
  - `boots` (5)
- Os peitorais existentes foram mantidos e mapeados como `chest`.

#### 3. Loja e Inventario Evoluidos
- `ShopTab` agora filtra por todos os tipos de equipamento.
- Cada item exibe:
  - peso
  - requisitos
  - scaling (quando aplicavel)
- `InventoryTab` com paperdoll ampliado para os novos slots.
- Itens sem requisito atendido sao destacados e bloqueados para equipar.

#### 4. Onboarding de Personagem (Build Inicial)
- A criacao de personagem agora inclui distribuicao inicial de atributos.
- Regras de equilibrio:
  - 8 pontos obrigatorios no onboarding
  - maximo 4 pontos por atributo
  - validacao no frontend e backend
- Presets adicionados:
  - Balanceada, Forca, Destreza, Tanque
- Previa de status final antes de confirmar personagem.

#### 5. Integracao Google AI / Nano Banana
- A chave da API foi validada e os modelos disponiveis foram listados com sucesso.
- Script criado: `scripts/generate-class-images.mjs`
  - gera retratos de classes em `public/classes`
- Conexao na tela de classe pronta (`imageSrc` por classe).
- Falha atual:
  - geracao de imagem bloqueada por quota (`HTTP 429 RESOURCE_EXHAUSTED`).
- `CharacterPortrait` agora tem fallback robusto para emoji quando a imagem nao existe.

### Validacao Tecnica
- `npx tsc --noEmit` passando nas alteracoes recentes.
- Lint do projeto segue com erros preexistentes em varios arquivos antigos (nao iniciado nesta etapa).

### Estado Atual
- Base pronta para expansao de RPG Souls-like com onboarding competitivo.
- Falta apenas liberar quota/billing no Google AI para gerar automaticamente os retratos das classes.

Projeto pronto para continuidade e balanceamento fino de progressao.
